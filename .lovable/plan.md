## 현재 문제\n\n`autoStopIdleChannels` cron은 GCP가 AWAITING_INPUT(RTMP 끊김)일 때만 종료합니다. 송출자가 OBS를 끄지 않으면 시청자 0명이어도 무한 송출. 실제 기록에 37시간, 167시간 세션 있음.\n\n## 목표 동작\n\n조건: is_live + 시청자 ≤ 2 + 송출시간 > 3시간 → 송출자에게 알림+모달 표시 \"계속 스트리밍 중이신가요?\"  \n[계속] 클릭 시 3시간 연장. 10분 무응답이면 자동 종료.\n\n## 구현\n\n### 1. DB (channels 컬럼 5개 추가, 모두 기본값)\n- auto_stop_max_minutes int DEFAULT 180\n- low_viewer_threshold int DEFAULT 2\n- keepalive_grace_minutes int DEFAULT 10\n- keepalive_prompt_sent_at timestamptz\n- keepalive_confirmed_at timestamptz\n\n### 2. Edge Function (`live-stream`)\n\n**기존 autoStopIdleChannels(2분 cron)에 로직 추가** — RTMP 살아있어도 채널 순회:\n\n- elapsed = now − GREATEST(live_started_at, keepalive_confirmed_at)\n- viewers = viewer_presence count (last_seen_at > now-2분)\n- IF elapsed > max_minutes AND viewers ≤ threshold:\n  - 프롬프트 미발송 → notifications INSERT + keepalive_prompt_sent_at=now()\n  - 프롬프트 발송 후 grace_minutes 초과 → stopOne(reason='무응답 자동 종료', end_reason='auto_unattended')\n\n**새 액션 confirmKeepalive** (인증 + owner 검증):\n- keepalive_confirmed_at=now(), keepalive_prompt_sent_at=NULL\n\n### 3. 프론트엔드\n\n- **송출자 모달** (BroadcasterControlPanel): keepalive_prompt_sent_at이 confirmed_at보다 신규일 때 표시. [계속 송출]/[지금 종료] 버튼, 카운트다운. 기존 Realtime 구독으로 즉시 반영.\n- **NotificationBell**: 새 type 'live_keepalive_prompt' (기존 인프라 그대로 사용, 링크 /my-channel).\n- **자동 종료 알림**: 기존 notify_live_lifecycle 트리거가 gcp_last_error 변경으로 송출자에 푸시.\n\n## 보호 장치\n\n- 시청자 3명 이상이면 프롬프트 안 뜸 (정상 예배 방해 X)\n- [계속] 한 번이면 다음 3시간 동안 재프롬프트 없음\n- 관리자 강제 종료와 독립 경로\n- 기본값으로 출시, 채널별 설정 UI는 추후\n\n## 테스트 시나리오\n\n1. 시청자 0 + 3h 1s → 알림+모달 → [계속] → 3h 연장\n2. 시청자 0 + 3h + 10분 11초 무응답 → 자동 종료\n3. 시청자 5 + 5h → 프롬프트 안 뜸\n</parameter>
